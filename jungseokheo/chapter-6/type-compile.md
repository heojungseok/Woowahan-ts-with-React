# Q1. 컴파일? 런타임?

A.
> 컴파일: 고수준 언어(인간이 이해하는 언어)에서 저수준 언어(기계가 이해하는 언어)로 변환하는 과정
> 런타임: 프로그램이 실제로 실행되고 있는 시점 또는 환경

여기서 타입 스크립트는 트랜스파일(Transpile)의 과정을 거친다. 

TypeScript → JavaScript 같은 수준의 언어 간 변환이 이뤄지고 타입 스크립트에서 컴파일러는 타입검사, Javascript로 변환 두 가지 역할을 수행한다.

타입스크립트는 컴파일타임에 타입을 검사하고 에러 발생 시 프로그램이 실행 되지 않아서 정적 타입 검사기라고도 한다.

런타임에는 순수한 JavaScript만 실행되므로 타입 정보가 없다.

# Q2. 타입스크립트 컴파일러 실행 5단계???

1️⃣ 스캐너 (Scanner)

.ts 파일의 코드를 읽어서 토큰화
토큰이란? 코드를 의미있는 최소 단위로 쪼개는 것
예: const name = "홍길동" → const, name, =, "홍길동" 으로 분리

2️⃣ 파서 (Parser)

토큰들을 기반으로 AST(추상 구문 트리) 생성
AST란? 코드의 구조를 트리 형태로 표현한 것
코드의 문법이 맞는지 확인하는 단계

3️⃣ 바인더 (Binder)

AST에 심볼(Symbol) 정보를 추가
심볼이란? 변수, 함수 등의 선언 정보
어디서 선언되고 어디서 사용되는지 연결하는 작업

4️⃣ 체커 (Checker)

AST + 심볼 정보로 타입 검사 수행
"이 변수는 number인데 string을 넣으려고 하네?" 같은 오류 찾기
타입 에러를 찾아내는 핵심 단계!

5️⃣ 이미터 (Emitter)

타입 검사가 끝나면 JavaScript 파일(.js) 생성
타입 정보를 모두 제거하고 순수 JS로 변환

요약하면:

`TS 코드 읽기 → 구조 파악 → 관계 연결 → 타입 검사 → JS 생성`
